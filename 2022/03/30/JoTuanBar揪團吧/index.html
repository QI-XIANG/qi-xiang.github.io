JoTuanBar 揪團吧

2022-03-30
2022-03-30
資管畢業專題

曾經的專題發想內容，經過些許整理後，成為過去留下的足跡…

緣起

你精打細算找到團購優惠卻湊不到人數嗎? 你非常喜歡玩桌遊卻找不到玩伴嗎? 你想去戶外探險卻找不到夥伴陪你瘋嗎? 那你可以試試"揪團吧(JoTuanBar)"，它是一款多元媒合APP，你可以在上面發起或參與各類型的揪團，快速找到適合你的他(們)。

傳統社群平台fb、dcard等，通常以結構較為鬆散的社團或是討論版來聚集擁有特定興趣、目的的人，但我們可以透過 JoTuanBar 揪團吧來改善這個問題，使用者可以在同個網站上一次搜尋到絕大多數感興趣的揪團內容，而不必再博覽各大討論版或參加複數社團。

應用範圍與情境

  1. 每個用戶都可成為系統中的節點 (已由LBRY影音平台證實可行)
  2. 註冊後，用戶可發起各類型的揪團
  3. 揪團發起審查自動化
  4. 用戶可自行建立群組進行後續討論 (群組存續時間會有不同限制)
  5. 用戶有等級上的不同
    (ex. 像eeclass一樣須完成特定的任務，獲取經驗值來升級)
  6. 商談合作夥伴 (提升特定活動的觸擊率)
  7. 廣告投放
  8. 自發Token回饋機制
  9. 用戶可創建公會招集志同道合的夥伴們，更快揪團成功

關於採用IPFS來分散式儲存資料

為什麼採用IPFS?

IPFS跟區塊鏈是不一樣的東西，由於它強調去中心化的特性，因此才容易被跟區塊鏈混為一談。

傳統網站的資料是由一個 Server 進行集中管理，但這樣會有個問題就是 Server 出狀況，網站就會變成離線狀態。

而在 IPFS 中，不存在中心化 Server 用來存放資料，它採用 P2P 網路的方式，資料由網路中的特定節點上傳，其它節點則可以協助保存資料副本，也就是說就算單一節點被攻擊下線，用戶端仍可經由其它節點獲得資料，網站也就不會出現離線的狀況。

  • 集中式儲存 VS. 分散式儲存

IPFS 內容定位

我們現在取得網路上資源時，大多都利用地址定位來取得，而 IPFS 則採內容定位的方式取得資料。

  • 地址定址

  • 內容定址

前述 P2P 網路中的節點在上傳資料時，會自動生產一組 hash 值，之後使用者可以利用這組 hash 值進行內容定位，跟單一或複數節點同時要資料。由於可以一次跟複數節點要資料的特性，比起跟傳統的單一中心化 Server 要資料快上許多。

  • 產生的 hash 值

但是 hash 值超不直覺，沒人能從字面上看懂，所以 IPFS 有提供可以將 hash 值 mapping 對應到一般人能夠看得懂的網址格式功能 : dnslink。

  • DNSLink

    https://docs.ipfs.io/concepts/dnslink/

IPFS 後臺管理

IPFS 有一套屬於自己的後台管理介面，可以看到資料進出速度、自己的節點 ID、IPFS 網路上的檔案內容、檔案的版本更新紀錄和分支狀況、網路上存活的節點。


(https://github.com/ipfs/ipfs-webui)

IPFS 應用案例

實例 : 土耳其封鎖維基百科事件

土耳其官方之所以封鎖維基百科，起因於英文版維基有個關於「由國家資助的恐怖主義」頁面，將土耳其政府名列其中，土耳其政府要求移除無果，決定全面封鎖維基百科。

ITFS 的官方 Twitter 帳號後表示，它們成功把維基百科網頁內容放到 IPFS 網路上，土耳其用戶因而可以突破政府封鎖，繼續正常地瀏覽網站、保持資訊傳遞暢通。

  • ITFS 在官方 Twitter 上的貼文

IPFS 願景

IPFS 未來的目標是超越 HTTP，讓現行的網頁內容可以被穩定、長久的存續下去。或許有一天，網頁不會再有404錯誤出現。
